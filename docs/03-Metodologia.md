
# Metodologia

A metodologia utilizada é o <b>Scrum</b>, aplicada tanto na resolução de problemas quanto no desenvolvimento do projeto. Para o controle do fluxo de atividades de cada membro da equipe, utiliza-se o <b>Kanban</b>. Nesse contexto, insere-se o ciclo de Sprint, no qual são realizadas entregas periódicas e reuniões focadas na melhoria contínua e no aprimoramento do projeto.

## Controle de Versão

A ferramenta de controle de versão adotada no projeto foi o
[Git](https://git-scm.com/), sendo que o [Github](https://github.com)
foi utilizado para hospedagem do repositório.

O projeto segue a seguinte convenção para o nome de branches:

- `main`: versão estável já testada do software
- `dev`: versão em testes do software
- 
Quanto à gerência de issues, o projeto adota a seguinte convenção para
etiquetas:

- `bug`: uma funcionalidade encontra-se com problemas
- `enhancement`: uma funcionalidade precisa ser melhorada
- `help wanted`: requer atenção especial ou contribuição externa

## Ferramentas

As ferramentas empregadas no projeto são:

| AMBIENTE                            | OBJETIVO                           |
|-------------------------------------|------------------------------------|
| Discord                             | Comunicação e reuniões da equipe   |
| WhatsApp                            | Comunicação rápida e informal      | 
| Git/Github                          | Entrega de trabalho                | 
| Github Projects                     | Acompanhamento do progresso Kanban | 
| Visual Studio Code                  | Desenvolvimento de códigos         | 
| Figma                               | Criação de interface e wireframes  |
| Azure                               | Hospedagem do site                 | 





As ferramentas utilizadas foram o Git e o GitHub, por sererm amplamente utilizados no mercado para versionamento e colaboração em código, garantindo o acompanhamento detalhado das mudanças e facilitando o trabalho simultâneo entre os integrantes. O GitHub Projects foi escolhido por sua integração direta com o repositório, dispensando o uso de soluções externas para gestão de tarefas e permitindo conectar facilmente issues, pull requests e o quadro Kanban.

Referente à comunicação, o Discord e o WhatsApp foram adotados para atender diferentes necessidades: O Discord é utilizado em reuniões mais formais e organizadas, enquanto o WhatsApp serve para interações rápidas e informais no cotidiano. Para prototipação, o Figma foi selecionado por ser uma plataforma colaborativa em nuvem, possibilitando que todos os membros acessem, visualizem e comentem os protótipos em tempo real, sem necessidade de instalação. Por fim, o Visual Studio Code se destaca como editor de código por ser leve, altamente personalizável e possuir integração nativa com o Git, contribuindo para um processo de desenvolvimento mais eficiente.
